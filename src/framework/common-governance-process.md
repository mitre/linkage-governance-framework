# Common Governance Process

The Common Governance Process is the operational heart of the framework, providing a structured way for investigators, requesting institutions, and dataset holders to work together on complex data linkages that would otherwise be slow, confusing, or infeasible. Organized into three stages — Intake, Screening, and Rule Harmonization — the process consolidates information into a single request package, coordinates reviews across dataset holders, and supports collaborative development of a set of rules that can safely govern a proposed linkage and resulting linked dataset.  
 
The Common Governance Process represents a departure from how governance decisions are made for linkage today by: 

- Consolidating multiple request forms into a single package for Dataset Holder review. 
- Using a Coordinating Body as the interface between the Investigator and Dataset Holders
- Synchronizing review processes across Dataset Holders to occur simultaneously and collaboratively
- Harmonizing governance rules across datasets toward a single rule set to govern the linkage implementation and resulting linked dataset 

While still a proof of concept, this process is designed to reduce governance bottlenecks, preserve each dataset holder’s authority, and ultimately help more highvalue linkages move from idea to implementation.
